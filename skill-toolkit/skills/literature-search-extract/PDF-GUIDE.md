# 本地 PDF 庫整合 — 使用說明（給使用者）

> 這是人讀說明文件。skill 的機讀規則本體在
> `references/search-sources.md` 的「Local PDF library」一節；兩者若有出入，以機讀規則為準。

## 這是什麼

literature-search-extract 除了上網檢索文獻，也能直接吃**你手上已有的論文 PDF 資料夾**：
把你的個人論文收藏當成一個檢索管道 (channel)，在裡面找到相關論文、逐節精讀、
萃取你要的資訊——因為讀得到全文，萃取結果會標成最高的 `[full]` 存取層級
（比網路上常見的「只讀得到摘要 `[abstract]`」可靠得多）。

## 怎麼觸發

對 Claude 說出「文獻需求 + 指向你的 PDF 資料夾」即可，例如：

- 「我的論文都放在 `<local-papers>/plasmonics/`，幫我從裡面找 SPP 傳播損耗的報導值，整理成參數表」
- 「用我 `~/Documents/refs` 資料夾裡的 PDF，比較這幾篇的 HLN 擬合方法」
- 也可以混用：「先用我本地的 PDF，不夠再上網補」——skill 預設就會這樣做（見下方「收藏偏誤」）。

## 執行時 skill 會做什麼（你會看到的行為）

1. **先盤點、後萃取**：掃描資料夾內所有 PDF，各讀第一頁辨識標題／作者／DOI，
   建一份「檔案路徑 ↔ 文獻識別碼」的小索引，然後只精讀跟你的問題有關的那幾篇。
2. **線上查核不會省略**：本地 PDF 只證明「內容長這樣」，不證明書目資訊正確。
   skill 仍會用 DOI 上 Crossref 核對引用欄位、查撤稿 (retraction)；如果你存的是
   preprint 版，會查有沒有已刊出版本並改引刊出版。
3. **收藏偏誤警示**：個人收藏反映的是你過去的閱讀軌跡，不等於該領域的全貌。
   在 standard／exhaustive 深度下，skill 會用網路管道補搜，並在 `search_trail`
   標明哪些結論**只有**你本地收藏支持——這是刻意設計的防偏誤機制，不是多此一舉。
4. **與 prism 併用**（若你的收藏已 ingest 進 prism）：先用 prism 排序找出最相關的
   節點，再回頭讀對應的 PDF 原檔做 `[full]` 級萃取；只靠 prism 的摘錄最多算 `[partial]`。

## 限制與注意事項

- PDF 逐次最多讀 20 頁，長論文會分段讀（自動處理，但特別長的書籍章節會慢一些）。
- 掃描檔（純圖片、無文字層）的 PDF 可能讀不出內容；遇到會如實回報，不會硬猜。
- 資料夾很大時（數百篇），盤點本身有成本——建議指到子資料夾，或先講清楚主題範圍。
- 檔名不影響辨識（skill 讀第一頁辨識論文身分），但識別碼缺失（無 DOI 的老文獻、
  技術報告）時會退回用書目字串比對，準確度略降並會註明。

## 驗收狀態

此整合於 2026-07-10 寫入規則，尚未經實際資料夾驗收（無 fixture PDF 可自動測試）。
你第一次用本地資料夾調用時就是驗收：若行為與本文件不符，請回報，屆時修規則並更新本文件。
